# Implement Next Slice

## Purpose
Ship a bounded improvement with validation and low waste.

## When To Use
- Executing a planned slice
- Turning a small design decision into code

## Required Checks
- Change only the files needed for the slice.
- Run the promised validation commands.
- Keep backend concerns outside gameplay modules.
- Leave notes for any deferred risk.

## Output Format
- Change summary
- Validation result
- Remaining risk

## Rollback / Failure Notes
If validation fails and the fix expands scope too far, revert to the last working slice and restate the plan.
