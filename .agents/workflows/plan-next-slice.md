# Plan Next Slice

## Purpose
Define the next smallest valuable change before implementation.

## When To Use
- Starting a new task
- After finishing a slice and choosing what comes next

## Required Checks
- The slice is independently reviewable.
- Mobile performance risk is known.
- Validation is named before coding starts.
- Scope fits existing architecture boundaries.

## Output Format
- Objective
- Why now
- Files
- Validation
- Risk / fallback

## Rollback / Failure Notes
If the slice cannot be validated quickly, split it again.
